# dash-drop

**Schema-in. Dashboard-out. In under 60 seconds.**

A schema-driven CRUD dashboard generator built with React + Vite + Node/Express + Prisma + PostgreSQL.
Drop in a JSON schema and get a full-featured admin dashboard with auth, search, filters, CSV export, and role-based access control — no boilerplate, no manual wiring.

---

## Features

- 🗂 **Schema-driven** — define entities and fields in `schema.config.json`, get a full dashboard automatically
- 🔐 **JWT Auth + RBAC** — Admin / Editor / Viewer roles out of the box
- 🔍 **Search & Filters** — full-text search + column filters on any field
- 📄 **Pagination** — server-side, configurable page size
- 📤 **CSV Export** — download filtered data as CSV
- 👁 **Column Visibility** — toggle columns per entity, persisted in localStorage
- ✅ **Inline Validation** — client + server-side error mapping
- 🔒 **Read-only Entities** — `"readOnly": true` hides CRUD buttons
- 🌙 **Dark Mode** — toggle, persisted in localStorage
- 🐳 **Docker Compose** — one-command local startup

## Quick Start (Docker)

```bash
git clone https://github.com/yourusername/dash-drop.git
cd dash-drop
cp .env.example .env          # optional — defaults work for local dev
docker compose up
```

- **Dashboard:** http://localhost:5173
- **API:** http://localhost:3001
- **Default login:** `admin` / `admin123`

## Schema Config

Edit `schema.config.json` to define your entities:

```json
{
  "entities": [
    {
      "name": "Contact",
      "icon": "Users",
      "fields": [
        { "name": "name", "type": "string", "required": true, "searchable": true },
        { "name": "email", "type": "string", "required": true },
        { "name": "status", "type": "enum", "options": ["Lead", "Customer"] }
      ]
    }
  ]
}
```

Supported field types: `string`, `text`, `number`, `boolean`, `date`, `enum`

## Deploy to Render

[![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy)

One-click deploy via `render.yaml` included in this repo. Spins up:
- PostgreSQL (free tier)
- API server (Node/Express + Prisma)
- Static client (React + Vite)

### Setup Steps

1. **Click the Deploy to Render button** above, or go to [render.com/deploy](https://render.com/deploy) and point it at this repo.
2. Render will auto-detect `render.yaml` and provision all three services.
3. Wait ~3-5 minutes for the initial build and database setup.
4. Open the **dash-drop-client** URL from your Render dashboard.
5. Log in with the default credentials: `admin` / `admin123`
6. *(Recommended)* Change the admin password immediately after first login.

> **Note:** Free tier services sleep after 15 minutes of inactivity. First request after sleep takes ~30 seconds to wake up.

### Environment Variables (auto-configured)

| Variable | Source |
|----------|--------|
| `DATABASE_URL` | Auto-wired from the Render PostgreSQL service |
| `JWT_SECRET` | Auto-generated by Render |
| `NODE_ENV` | Set to `production` |

For local development, all variables are documented in [`.env.example`](.env.example).  
Copy it with `cp .env.example .env` — the defaults work out of the box with Docker Compose.
