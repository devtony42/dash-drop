# Quick Start

Get a working dashboard in three commands:

  1. Clone the repository: git clone https://github.com/devtony42/dash-drop.git
  2. Enter the directory: cd dash-drop
  3. Start everything: docker compose up

Open http://localhost:3000 to see your dashboard with demo data (Products and Orders).

The server automatically generates the database schema from schema.config.json, runs migrations, and seeds sample data on first boot.

# Schema Configuration

Define entities in schema.config.json at the project root. Each entity becomes a database table, a set of REST API endpoints, and a full UI with table, forms, and filters.

{ "entities": [ { "name": "Product", "fields": [ { "name": "name", "type": "string", "required": true, "searchable": true }, { "name": "price", "type": "number", "required": true }, { "name": "category", "type": "enum", "options": ["Electronics", "Clothing", "Food"] }, { "name": "inStock", "type": "boolean", "default": true }, { "name": "description", "type": "text" } ] } ] }

Field Types

TypeRenders AsDescription
stringText inputShort text, supports searchable: true
numberNumber inputIntegers or decimals
booleanCheckboxTrue/false toggle
dateDate pickerDate selection
enumDropdownSelect from options array
textTextareaMulti-line text

Change the config and restart — the new entity appears automatically with full CRUD, search, and filtering.
